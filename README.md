# PhysiCymatics 🔬🎵

**音が、本物の物理で、目に見える形になるアプリ。**

音の周波数や音楽をならすと、板の上の砂や、皿の水に模様が生まれる——「サイマティクス」。
このアプリは、その模様を**ぜんぶ本物の物理方程式の計算**で表示します。見た目のための細工はゼロ。

## 5つの約束

1. **すべての模様は方程式の解**。「こう見えたらきれい」の調整はしない
2. 計算がまだ本物でない部分は、**紫のMOCK表示で自白**する
3. 特定の周波数（432Hzなど）の**優遇はしない**
4. すべての実行は記録され、**誰でも（AIでも）再現できる**
5. 美しさは**光の物理**（反射・屈折・きらめき）から来る。運動の物理には手を触れない

## はじめての実験（うえきさん用・やさしい版）🥣

**用意するもの**: 平らな皿（水深2cmくらい）、スピーカー（皿を上に乗せられるもの）、スマホ

1. 皿に水を入れて、スピーカーの上へ
2. **60Hzの音**を鳴らす（YouTubeで「60Hz tone」でOK）
3. 音量をゼロからゆっくり上げる

**確認チェック欄**（できたら✓）:
- [ ] ある音量から急に、水面に**格子模様**が出た（→出た音量の目盛をメモ）
- [ ] 模様の間隔は**だいたい9mm**？（ものさしを横に置いて写真）
- [ ] スマホのスローモーション（240fps）で撮ると、模様が**音の半分の速さ**でチカチカしてる？
- [ ] 音を**440Hz**に変えると：模様はずっと細かい光沢になり、もっと大きな音量が要る？

**結果の送り方**: このリポジトリの Issues → New issue → 「実験報告」を選んで、チェックと写真を貼るだけ。
うまくいかなくても大収穫です（計算とのズレそのものがデータ）。

## フォルダの地図

- `physics_pack/` — アプリが読む実データ（板のドレミ・水の閾値と波長・砂の窓・形）
- `oracles/` — 真実の審判たち（検証済みPythonコード。実装はこれらと一致しなければならない）
- `docs/` — 監査記録・設計文書・作業台帳（罠の図鑑つき）
- `ui_shell/` — UI/UXのプロトタイプ（React）と配線指示
- `reference/` → `core/` → アプリ本体（Phase 1→3で成長）
- `ledger/` — 記録ルーム。人もAIも、ここに足あとを残す
